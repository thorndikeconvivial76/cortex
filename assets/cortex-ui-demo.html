Cortex is a persistent memory layer for Claude Code. Every session starts with full context — across projects, machines, and days.

The Problem

Claude Code is extraordinary at coding. But every new session is a blank slate. No memory of last week's architecture decisions. No awareness of your preferences. No knowledge of the open threads from yesterday. You re-explain everything, every time.

How It Works

Dead simple. Invisibly powerful.

1
Install in 30 seconds
One command. Cortex installs, starts a background daemon, and wires itself into Claude Code automatically. No manual config. No setup wizard.
brew install cortex.memory
✓ Cortex is running. Open Claude Code to start.
2
Cortex learns silently
As you work, Cortex quietly captures the things worth remembering — architectural decisions, preferences, open problems, resolved bugs. No prompts, no interruptions.
3
Claude starts every session informed
Next session, Claude already knows your project. The stack, the decisions, the open threads. You start coding immediately instead of re-explaining context.
✓ 12 memories loaded for karnyx
✓ Last session: 2 days ago
✓ Open thread: Deepgram transcription latency
4
Works across all your machines
Memories sync quietly to the cloud. Open Claude Code on your second machine — same context, same knowledge, instantly in sync.

Security

Built for engineers who ask hard questions

🔒
Local by default
All memories stay on your machine. Nothing leaves unless you explicitly enable cloud sync. Your code, decisions, and context are yours alone.
👁
Full transparency
Fully open source. Every line of code is readable on GitHub before you install. No hidden behavior, no surprise network calls, no telemetry ever.
🗑
Clean uninstall
One command removes everything — daemon, database, config files, and all .cortex project markers. Zero footprint left behind.
🌐
Localhost only
The MCP server listens on localhost:7433 exclusively. It is not exposed to your local network or the internet under any circumstances.
🧹
No sensitive storage
Cortex never stores code, API keys, credentials, or file contents. Only structured memory records about decisions and context.
✋
You control everything
cortex show reveals all stored memories. cortex clear deletes them. cortex export lets you take your data anywhere. No lock-in.
